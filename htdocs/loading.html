Communications Server Linux STREAMS (CSLiS)

Loading CSLiS

Once you have built CSLiS you need to load it into memory in order to execute STREAMS drivers. There are two main ways to load the driver into memory. First, the Communications Server will load the STREAMS drivers if the drivers are not already loaded when the "sna start" is issued. Secondly, the STREAMS drivers can be loaded using modprobe command. k

CSLiS as Loadable Module

The build process should have copied the file streams.ko to the directory /lib/modules/kernel-version/misc. Where kernel-version is something like 4.12.14-22.7

This means that to load CSLiS you simply type in "modprobe streams". On SLES 12 and 15, you will need to issue "modprobe --allow-unsupported streams". However, the recommended method for starting CSLiS is to let the Communications Server start streams, sna start.

When CSLiS loads it will print a series of messages in the system message log showing the names and major device numbers of all STREAMS drivers and pushable modules loaded with it. It will also print some version information about itself, including the version of the kernel for which it has been compiled.

Making /dev Entries

A "normal" CSLiS build will have made a program named "makenodes" and copied it to /usr/sbin/strmakenodes. Before you can use any of your STREAMS drivers you must run that program. Simply enter "/usr/sbin/strmakenodes" and the program will make all of the STREAMS related entries in the /dev directory.

You do not need to do this every time you rebuild CSLiS. This only needs to be done if you add "node" entries to the STREAMS Config file or if you do something that would cause the major device numbers of STREAMS files to change.

The buildLiS script's "make install" target of CSLiS runs strmakenodes automatically and is the recommended method of making the /dev entries.

It is now possible for STREAMS drivers to make their own device nodes by calling the routine lis_mknod(). This is useful for loadable STREAMS drivers.
